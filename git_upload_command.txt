> git-hub에서 repository 생성

> .ignore.txt 생성 > 저장

> main에 1차 업로드 (main에 파일이 하나라도 있어야 branch 생성이 가능함)
    git add .ignore.txt
    git commit -m ""
    git push origin

> 브랜치 생성
    git branch 브랜치명

> 브랜치로 이동
    git checkout 브랜치명

> 작업한 항목 업로드 ㅡ 가능한 main에 바로 업로드(push)하지 않도록
    git add .
    git commit -m ""
    git push origin 브렌치명

> 작업이 끝난 브랜치 merge (main <- 브랜치명)
1) pull request 이용
2) git merge 이용
    git branch main
    git merge 브랜치명
    git push origin main

> 작업이 끝난 브랜치 제거
    git branche -d 브랜치명   ㅡ 로컬 브랜치 삭제
    git push origin -d 브랜치명 ㅡ 원격 브랜치 삭제
